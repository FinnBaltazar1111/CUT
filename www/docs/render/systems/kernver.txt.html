Kernel Versions

A description of kernel max rollback versions (kernvers) and the exploits available for them

Section 1 - Technical overview
The kernel version rollback protection system, referred to as the kernver in the
Firmware
, is a 4-byte section of memory in the
GSC
that is used to determine the minimum signed ChromeOS version that can be loaded.
TL;DR it's Google's system to prevent downgrading to utilize critical exploits.
Section 2 - Kernvers and exploits
The folowing is a list of kernel versions, their minimum downgrade version, and the associated exploits, respectively. If a ChromeOS version is prefaced with a 'v', the listed ChromeOS version is vulnurable to that exploit, as well as any below it.
